Description du système - SvxLink / SvxReflector

SvxReflector (anciennement également appelé "Östkustlänken") est un réseau national de répéteurs analogiques suédois, qui est relié avec l'aide de SvxLink , généralement sur Internet.

Dans le réseau, les groupes de discussion permettaient de séparer le trafic. Un groupe de discussion peut être assimilé à un "canal" logique.

  • Un répéteur peut accueillir (surveiller) plusieurs groupes vocaux. Le groupe de conversation adapté à chaque répéteur est indiqué sur la page d'accueil de SvxPortalen.
  • Un groupe de voix peut être sélectionné manuellement par l'utilisateur local, en utilisant les commandes DTMF ou Subton (CTCSS).
  • Le répéteur peut également avoir un groupe de numéros par défaut local, qui est activé automatiquement au démarrage local (si l'utilisateur ne fait pas de sélection active).

D'un autre côté, un répéteur ne peut jamais être actif sur plus d'un groupe vocal à la fois.

Le trafic entrant sur un groupe de voix surveillé démarrera automatiquement le répéteur, mais ne peut normalement pas entrer par effraction pendant un QSO en cours.

Les répéteurs et les nœuds Simplex du réseau surveillent de manière statique:

  • Groupe de discours 240, qui couvre toute la Suède.
  • Propre groupe de numéros de district - par exemple 2405 pour SM5.
  • Répéteurs et districts à proximité possibles.
  • Possiblement propre groupe de numéros locaux - par exemple 240515 pour Eskilstuna. Celui-ci peut alors être utilisé pour "adresser" uniquement ce répéteur.
  • Eventuellement groupe de numéros de cluster - par exemple 24078 pour SK7RFL + SK7RN à terre ou 24021 pour la «ligne Kalix».

L'ensemble du système peut être surveillé en temps réel via le portail Svx (ce site Web).

Les dernières informations peuvent être trouvées sur une liste de diffusion: https://svxreflector-sm.groups.io/

Choix du groupe de discussion

Si vous souhaitez utiliser un groupe vocal spécifique, il peut être sélectionné de deux manières - avec DTMF ou Subton (CTCSS).

Si aucun trafic ne se produit sur le groupe de conversation sélectionné, le système passe en mode veille après 30 secondes (réglable par sysop) et attend l'activité sur les groupes vocaux surveillés. Un petit "FLIRRP" est entendu lorsque le répéteur ou le nœud simplex revient à sa position par défaut. Sur SK7RFL, SK7RN et SK5BN, cela se produit toujours lors de la fermeture du répéteur.

DTMF

Pour activer le groupe vocal 2405 avec DTMF, appuyez sur: 

912405#

Le répéteur annoncera qu'il passera au groupe de conversation 2405 lorsque vous relâchez PTT.

Subton (CTCSS)

Une liste des groupes de discussion qui peuvent être activés avec Subton peut être trouvée ici sur le SvxPortal .

Pour activer par exemple le groupe de conversation 2405 avec Subton, CTCSS, sélectionnez Subton 91.5Hz. Après la première session de diffusion, Tepeatern annonce à quel groupe de parole il basculera lorsque vous relâchez PTT.

L'activation avec une sous-tonalité ne peut avoir lieu qu'à partir du mode veille du répéteur, c'est-à-dire pas pendant le QSO en cours.

Conseil

Programmez plusieurs canaux mémoire avec différentes sous-tonalités, afin de pouvoir activer directement le nœud Répéteur / Simplex avec le bon groupe de conversation.

Pour les nœuds qui ont un Subton qui n'est connecté à aucun groupe de conversation, colonne locale dans la liste, les services sur le nœud peuvent être utilisés sans qu'aucun groupe de voix ne soit connecté. Il peut être pratique pour utiliser divers modules dans SvxLink tels que parot, EchoLink, messagerie vocale, etc. Il permet également d'avoir des QSO locaux sur le canal, sans qu'aucun groupe de parole ne soit activé de l'extérieur.

De la même manière que lorsqu'un groupe de voix est actif, il est nécessaire que le canal soit inactif pendant un certain temps avant que la connexion des groupes de voix puisse avoir lieu, à la fois à distance et avec une sous-tonalité. L'heure est réglable par sysop, mais est par défaut de 30 secondes.

Cependant, il est toujours possible d'activer / de changer de groupe de conversation avec des commandes DTMF à tout moment.

QSY

Lors d'un QSO sur un groupe de parole, il est possible de déplacer tous les nœuds actifs vers un autre groupe de parole. Le nœud actif est le nœud qui a eu une activité locale depuis que le talkgroup a été activé, c'est-à-dire que le mur antibruit s'est ouvert au moins une fois.

La fonction QSY est contrôlée par des commandes DTMF.

92# QSY à un groupe de nombres aléatoires qui est automatiquement sélectionné par le serveur de réflecteur
92nnn# QSY au groupe de parole nnn
93# Suivez le dernier QSY

QSY pour un groupe de parole temporaire est utilisé de manière appropriée de telle sorte que les appels vers un début soient effectués sur un groupe de parole plus large, par exemple 240 pour l'ensemble de la Suède. Lorsque tous les participants ont appelé sur le groupe de conversation, la commande QSY ( 92 # ) est utilisée - pour déplacer tous les nœuds actifs vers le groupe vocal temporaire. Le groupe de numéros plus large sera alors à nouveau gratuit pour les appels.

Un QSY automatique est également effectué sur le groupe de conversation 240, après 5 minutes.

QSY à un groupe de voix spécifié peut être utilisé pour ajouter plus de nœuds à un QSO en cours. Supposons qu'un QSO a démarré sur le groupe de conversation 2403, c'est-à-dire SM3. Sous le QSO, vous souhaitez également développer des nœuds dans SM5, c'est-à-dire le groupe de numéros 2405. La commande 922405 # peut alors être utilisée pour cela. Tous les nœuds actuellement actifs sur 2403 seront déplacés vers 2405 et tous les nœuds qui surveillent 2405 se connecteront au groupe vocal.

Dans les nouvelles versions de SvxLink, vous pouvez très facilement suivre un QSY. Lorsque QSY se produit, le répéteur dit " QSY 24099nn en attente ". Si vous appuyez sur PTT dans les 15 secondes environ, le répéteur suivra ce QSY. Sinon, le répéteur dit " QSY [24099nn] ignoré ".

Un nœud passif peut également être fait manuellement pour suivre le dernier QSY avec la commande 93 # . Cela peut être utile si vous n'avez pas eu le temps d'envoyer avant le QSY ou si vous avez eu le temps d'appuyer sur PTT dans les 15 secondes après QSY.

Notez la différence entre QSY et le choix local de TG.

  • 922405 # déplace tous les nœuds actifs vers le groupe de conversation 2405 (QSY).
  • 912405 # déplace uniquement le nœud local vers le groupe de conversation 2405 (sélection du groupe de voix).

Informations supplémentaires

Le mappage entre les sous-tons et les groupes de conversation est conforme à la liste SSA pour CTCSS. La deuxième tonalité de chaque district (Secondaire 1) est mappée au groupe de conversation de ce district.

Si vous souhaitez déconnecter un groupe de conversation directement, sans attendre 30 secondes, vous pouvez appuyer sur 910 # (TG0). Bon à utiliser si vous choisissez le mauvais groupe de discours.

Vous ne pouvez pas activer plusieurs groupes de conversation en même temps.

Si vous avez activé un groupe de discussion spécifique, par exemple TG2403, le trafic provenant d'un autre groupe de discussion sera bloqué jusqu'à ce que le trafic soit arrêté sur TG2403 et que 30 secondes se soient écoulées.

Le SvxReflector n'est pas connecté à DMR / Brandmeister, mais utilise des noms et des numéros similaires sur le groupe de discussion.
